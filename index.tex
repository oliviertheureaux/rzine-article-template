\subsection{Template Quarto}\label{template-quarto}

Un modèle de mise en page Quarto\footnote{\url{https://quarto.org}} est
\href{https://github.com/rzine-reviews/rzine-article-template}{mis à
disposition sur GitHub}.

Pour le récupérer en ligne de commande :

\begin{Shaded}
\begin{Highlighting}[]
\ExtensionTok{quarto}\NormalTok{ use template rzine{-}reviews/rzine{-}article{-}template}
\end{Highlighting}
\end{Shaded}

Cela crée un répertoire sur votre machine contenant un fichier Quarto
\texttt{index.qmd}, prêt à être complété.

\subsection{Métadonnées}\label{muxe9tadonnuxe9es}

Il n'y a pas d'en-tête (YAML) dans le fichier source de l'article
(\texttt{index.qmd}). Les métadonnées générales de l'article sont à
complétées dans le fichier \texttt{\_quarto.yml} situé à la racine du
répertoire. Renseignez-y le titre, sous-titre, auteur, affiliations,
résumé, mots-clés\ldots{}

\section{Annexes de l'article}\label{annexes-de-larticle}

\subsection{Données}\label{donnuxe9es}

Tout·e lecteur·ice doit être en mesure de reproduire la démonstration de
l'article sans contrainte de réutilisation. L'intégralité des données
utilisées doit obligatoirement être :

\begin{itemize}
\tightlist
\item
  Correctement présentées dans l'article
\item
  Libre d'utilisation et de diffusion
\item
  Mises à disposition des lecteur·rice·s, même si ces données sont déjà
  en libre accès
\item
  Décrites par des métadonnées (\texttt{readme.txt})
\item
  Respectueuses du règlement général sur la protection des données
\end{itemize}

\textbf{Toutes les données utilisées dans l'article devront être
stockées dans le répertoire \texttt{data/}.}
